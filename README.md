# Model Distillation Guide

## Building a High-Quality Student LLM from Teacher Models

---

# Overview

Model distillation is the process of transferring the knowledge of a large language model (the **Teacher**) into a much smaller model (the **Student**).

Instead of training from trillions of tokens from scratch, the student learns by imitating high-quality responses generated by the teacher.

```
                Teacher Model
                     │
                     ▼
          Generate High-Quality Responses
                     │
                     ▼
          Distillation Dataset (JSONL)
                     │
                     ▼
          Fine-tune Student Model
                     │
                     ▼
             Small Fast Local Model
```

---

# Phase 1 — Choose the Teacher

Select one or more high-quality teacher models.

Examples:

* Qwen3-8B
* Qwen3-32B
* DeepSeek-R1
* Gemma 3
* Phi-4
* Llama

Multiple teachers generally produce better datasets than a single teacher.

---

# Phase 2 — Choose the Student

Typical student models:

| Model   | Parameters |
| ------- | ---------: |
| SmolLM2 |       135M |
| SmolLM2 |       360M |
| Qwen2.5 |       0.5B |
| Qwen2.5 |       1.5B |
| Gemma   |         1B |
| Phi     |       Mini |

Choose the smallest model that satisfies your deployment requirements.

# Phase 3 — Build a Prompt Corpus

The prompt corpus determines what the student will learn.

Do **not** use only coding prompts.

Cover many domains.

Recommended categories:

* Programming
* Algorithms
* Mathematics
* Physics
* Chemistry
* Biology
* Linux
* Networking
* Cyber Security
* Databases
* AI
* Machine Learning
* Writing
* Summarization
* Translation
* Reasoning
* Logic
* Software Engineering
* DevOps
* Cloud
* System Design

Example:

```json
{
  "prompt": "Write a Go HTTP server."
}
```

# Phase 5 — Generate Teacher Responses

For every prompt:

```
Prompt
   │
   ▼
Teacher
   │
   ▼
Response
```

Example:

Prompt

```
Explain TCP.
```

Teacher Response

```
TCP is a reliable transport protocol...
```

Save every interaction.

---

# Phase 6 — Multi-Teacher Distillation (Recommended)

Instead of using one teacher:

```
Prompt
   │
   ├── Teacher A
   ├── Teacher B
   ├── Teacher C
   └── Teacher D
          │
          ▼
Compare Outputs
          │
          ▼
Choose Best
```

Advantages:

* Better factual accuracy
* Better coding quality
* Better reasoning
* Reduced hallucinations

---

# Phase 7 — Quality Filtering

Never keep every response.

Remove:

* Incorrect answers
* Hallucinations
* Repetitive answers
* Empty responses
* Very short answers
* Truncated responses
* Broken code
* Invalid JSON
* Duplicate samples

Good filtering often improves quality more than adding more data.

---

# Phase 8 — Optional Automatic Scoring

Score each answer using criteria such as:

* Correctness
* Completeness
* Readability
* Structure
* Reasoning quality
* Code compilation (for programming)
* Mathematical correctness

Discard low-scoring examples.

---

# Phase 9 — Create the Distillation Dataset

Example JSONL:

```json
{
  "messages":[
    {
      "role":"user",
      "content":"Write a binary search."
    },
    {
      "role":"assistant",
      "content":"..."
    }
  ]
}
```

One conversation per line.

---

# Phase 10 — Split the Dataset

Typical split:

* Training: 90%
* Validation: 5%
* Test: 5%

Never train on the validation or test sets.

---

# Phase 11 — Train the Student

Common libraries:

* Transformers
* Accelerate
* TRL
* PEFT
* Unsloth

Train with supervised fine-tuning on the distilled dataset.

---

# Phase 12 — Evaluate

Evaluate on unseen prompts.

Measure:

* Coding accuracy
* Math accuracy
* Reasoning
* Hallucination rate
* Latency
* Memory usage
* Token throughput

---

# Phase 13 — Improve

Repeat the process:

```
More Prompts
      │
      ▼
Better Teacher Responses
      │
      ▼
Better Dataset
      │
      ▼
Retrain Student
```

Every iteration generally improves the student if the new data is high quality.

# Best Practices

* Use multiple teacher models when possible.
* Prefer diverse prompts over repetitive ones.
* Keep a separate evaluation set.
* Remove duplicates.
* Filter aggressively for quality.
* Include code, STEM, reasoning, and general tasks.
* Track dataset versions for reproducibility.
* Retrain periodically as your dataset improves.
